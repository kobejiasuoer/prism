# Prism 股票参数与分析体系评估报告

Date: 2026-04-23
Reviewer: Codex
Scope: `stock-analyzer` + `packages/screener` + `apps/control-panel` + 历史样本 + GitHub 量化系统对标

## 1. 执行结论

Prism 当前不是一个通用型量化投资平台，而是一个偏 A 股短线战术决策的研究系统。

更准确地说：

- `stock-analyzer` 更像自选股 / 持仓的盘中规则化决策器。
- `packages/screener` 更像进攻型机会发现与盘中确认引擎。
- `control-panel` 负责把两个引擎的结论收束成操作视图。

从专业投资研究视角看，Prism 目前的优势在于“战术执行约束”而不在于“统计验证过的长期 alpha 研究”。

系统已经具备下列优点：

- 会主动区分环境强弱，而不是只看个股。
- 会限制追涨型 setup，避免单票高分直接放行。
- 会区分早盘结论和午盘确认，降低盘中误判。
- 会输出明确的触发位、回避位、止损位和轻仓建议。

但系统也有明显短板：

- 参数体系仍以硬编码经验阈值为主，缺少统一因子字典。
- 缺少完整回测、walk-forward、样本外验证和参数稳定性分析。
- 缺少组合层风险管理、仓位分配和风格暴露控制。
- 对基本面、盈利质量、预期修正、行业周期的分析仍然偏浅。
- `watchlist` 的技术评分依赖外部 `backtest.py`，公开仓库当前不可完整复核。

结论上，我认为 Prism 适合被定位为：

> 一个偏短线 / 题材 / 趋势 / 资金共振的 A 股战术决策系统，
> 而不是已经完成研究闭环的成熟量化投资平台。

## 3. Prism 现在到底在分析什么

### 3.1 `stock-analyzer` 的分析对象

`stock-analyzer` 当前主要分析固定自选股 / 持仓票，输出的是规则化快照，而不是深度研究报告。

它主要处理这些参数：

- 技术分：来自 `240` 分钟 K 线、MACD、KDJ、布林带、均线、价格区间位置。
- 资金流：东方财富主力净流入 / 连续流入 / 连续流出。
- 事件偏向：新闻、公告的正负面归类。
- 估值与质量：PE、PB、ROE。
- 行业相对强弱：个股涨跌与行业涨跌差。
- 交易位：支撑、压力、止损、盘中触发线。

系统最终给出：

- `action`
- `position`
- `hard_flags`
- `watch_points`
- `support / resistance / stop_loss`
- `intraday_triggers`

这说明它的核心任务是：

> 为已关注个股给出盘中和次日的纪律化处理建议。

### 3.2 `screener` 的分析对象

`packages/screener` 分析的是“进攻型机会池”，不是全市场通用股票研究。

它的真实分析链路分为四层：

1. 股票池约束
2. 初筛打分
3. 二次筛选
4. 午盘确认

#### 第一层：股票池约束

系统先用 `沪深300 + 中证500` 构造进攻池，并排除明显防守风格行业。

这意味着 Prism 当前默认追求的是：

- 主板
- 较高流动性
- 中大盘中偏活跃部分
- 非防守型方向

所以它并没有覆盖：

- 全 A 股
- 创业板 / 科创板核心高弹性群体
- 小票主题扩散早期
- 价值股 / 红利股 / 低波组合

#### 第二层：初筛参数

`scan.py` 初筛核心看的是：

- 成交额
- 换手率
- 日内振幅
- 20 日 K 线位置
- MA5 / MA10 / MA20 多头结构
- 放量程度
- 当日涨幅

这是典型的短线战术选股参数，而不是完整的多因子股票研究。

#### 第三层：二次筛选参数

`ai_screening.py` 明显比初筛更成熟，它会继续处理：

- 流动性是否足够执行
- 当日资金是否确认
- 行业 / 题材是否为主线
- 主题持续性是否增强或衰减
- PE / ROE 是否错配
- 公告风险是否限制执行
- setup 是否属于追涨型还是回踩型
- 当前市场环境是否允许开仓

最终它不会只给“分数”，还会给：

- `consistency`
- `execution_quality`
- `setup_type`
- `entry_plan`
- `tier A/B/C`

这一步的本质不是“预测涨跌”，而是：

> 判断今天这只票能不能做，应该怎么做，做到什么程度。

#### 第四层：午盘确认

`midday_verify.py` 会拿晨间候选和盘中状态对比，检查：

- 分数是否大幅下降
- 涨幅是否回落
- 主力资金是否延续
- 是否跌回 MA5 / MA10 / MA20
- 是否跌破晨间失效位
- 题材是否还在主线内

这一层很接近职业交易中的“盘中承接复核”，说明 Prism 的设计理念确实偏交易执行，而不是静态选股。

## 4. Prism 当前参数体系的优点

### 4.1 优点一：环境过滤意识很强

Prism 最值得肯定的一点，是它没有把“个股强”直接等同于“市场可做”。

系统会同时看：

- 全池环境
- 候选环境
- 二者差值

然后再决定阀门 `on / limited / off`。

这比很多只会筛热点票的系统成熟得多，因为它能识别：

- 候选票很强，但市场整体并不支持追涨
- 当前只能观察，不能执行

这个设计是正确的，应该保留并继续增强。

### 4.2 优点二：有明确的“过热惩罚”意识

系统会对以下行为主动降权：

- 高位大涨
- 高换手爆量
- 接近涨停
- 趋势票脱离支撑过远

这体现出很明确的交易纪律：

> Prism 不是单纯偏爱强势股，而是试图筛掉高潮末端。

这一点对 A 股尤其重要。

### 4.3 优点三：输出的是“执行计划”而不是空泛结论

很多系统只给“推荐 / 不推荐”，Prism 则进一步给出：

- 触发条件
- 回踩观察位
- 失效位
- 建议仓位
- 回避条件

这对真实交易非常有价值，因为它把“分析”转成了“可执行纪律”。

### 4.4 优点四：公告和事件风险有进入主流程

系统已经能把减持、诉讼、处罚、业绩预警等事件纳入筛选，不是纯技术面系统。这一步虽然还不深，但方向是对的。

## 5. Prism 当前参数体系的主要问题

### 5.1 问题一：参数是规则堆叠，不是可验证因子体系

现在大部分判断是：

- `if amount_yi < 4: exclude`
- `if pe >= 95 and roe < 5: exclude`
- `if score < 70: midday fail`
- `if quality < 4: caution`

这种方式的优点是可解释、容易上线；缺点是：

- 阈值是否稳定，不知道
- 参数之间是否重复计分，不清楚
- 哪些规则真正贡献了 alpha，不清楚
- 是否只对最近这段市场有效，不知道

换句话说，现在参数更像“经验化规则”，而不是“研究证明过的因子”。

### 5.2 问题二：基本面层明显偏薄

当前基本面主要用到了：

- PE
- PB
- ROE
- 毛利率

这在短线系统里够用作排雷，但远远不足以支撑专业股票研究。

缺失较明显的包括：

- 营收 / 利润增速
- 预期修正
- 自由现金流
- 资产负债质量
- 周期行业景气位置
- 行业相对估值分位
- 一致预期变化

所以 Prism 目前不适合被描述成“对股票进行了深入基本面分析”。

### 5.3 问题三：没有组合层

当前系统几乎所有判断都停留在单票层面：

- 这只票能不能做
- 做的话轻仓还是观察

但成熟投资系统还需要处理：

- 行业暴露
- 单主题拥挤度
- 单票 / 单日风险预算
- 组合相关性
- 风格偏移
- 回撤后的自动降风险

没有这一层，系统再聪明，也更像“候选票助手”而不是“投资系统”。

### 5.4 问题四：回测与样本外验证不足

目前仓库里能看到大量历史产物，但看不到一套完整、标准化、可重复的研究验证链路来回答：

- 这些规则过去几年胜率如何
- 在牛市 / 震荡 / 退潮期是否稳定
- 环境阀门是否真的提升了收益回撤比
- 午盘确认是否提升了出手质量
- 过热惩罚是否减少了回撤还是错杀了主升浪

没有这些验证，参数只能叫“有经验”，不能叫“有证据”。

### 5.5 问题五：数据时点一致性仍有风险

自选股样本里已经出现：

- 价格是当日 11:30
- 资金却只更新到前一日

虽然系统会提示 `flow_unconfirmed`，这是好事，但当前只是“提醒”，不是完整的权重折减框架。

这会带来一个真实风险：

> 盘中价格已经变化，但确认信号仍引用滞后的资金状态。

### 5.6 问题六：公开仓库存在一处可复核性缺口

`stock-analyzer/scripts/fetch.py` 技术分通过动态导入 `backtest.py` 计算，但公开仓库当前未找到该文件。

这意味着：

- 技术评分逻辑无法被完整复核
- 评分变化是否可靠，外部读者不能确认
- 维护者未来回看时也不够稳

这个问题不一定影响你本地系统运行，但会影响仓库级可审计性。

## 6. 和先进量化系统的差距

### 6.1 和 Qlib 的差距

Qlib 的核心不是某个选股规则，而是完整研究闭环：

- 数据处理
- 因子挖掘
- 模型训练
- 回测评估
- 风险建模
- 组合优化
- 线上服务

Prism 当前只强在中后段：

- 规则筛选
- 执行约束
- 操作展示

前半段的研究层明显不够。

### 6.2 和 vectorbt 的差距

vectorbt 的强项是：

- 大规模参数扫描
- walk-forward
- 多参数热力图
- 收益 / 回撤 / trade-level 分析
- 对策略稳定性的快速验证

Prism 现在很缺这种“参数为什么成立”的验证层。

### 6.3 和 backtrader / RQAlpha 的差距

这类系统的强项是：

- 交易成本建模
- 滑点
- 仓位管理
- 订单生命周期
- 组合回测
- 多标的同步运行

Prism 现在更偏“信号和纪律”，还没有真正进入“组合交易系统”的阶段。

## 7. 我对 Prism 当前定位的专业判断

如果按不同标准打分，我会这样看：

- 作为短线战术辅助系统：中上
- 作为规则化风控执行系统：中上
- 作为股票深度研究系统：中等偏下
- 作为成熟量化平台：当前不够

这不是否定，而是更准确的定位。

Prism 现在真正擅长的是：

- 找出相对活跃、相对强势、资金有承接的候选
- 在环境不配合时主动关闭进攻阀门
- 把盘中执行纪律结构化输出

Prism 现在还不擅长的是：

- 证明参数长期有效
- 做组合级资本配置
- 做中长期基本面研究
- 做因子归因和规则有效性归因

## 8. 改造建议

### 8.1 第一优先级：把参数整理成标准因子表

建议先把所有当前参数统一命名，沉淀成一个标准因子字典，例如：

- `liquidity_amount_yi`
- `turnover_ratio`
- `price_position_20d`
- `volume_expansion_ratio`
- `capital_flow_today_yi`
- `capital_flow_consecutive_days`
- `theme_persistence_score`
- `valuation_pe_ttm`
- `quality_roe`
- `event_risk_score`
- `execution_quality_score`
- `consistency_score`

每个字段都要写明：

- 定义
- 数据源
- 更新频率
- 缺失处理
- 是否可用于回测

没有这一层，后面的优化会一直陷入“改规则但说不清”的状态。

### 8.2 第二优先级：给现有规则补历史验证

建议不要急着上更复杂模型，而是先验证现在的规则值不值得保留。

建议最先测试：

- `execution_gate` 是否显著改善收益回撤比
- `midday_verify` 是否提升出手质量
- `overheat_penalty` 是否减少高潮回撤
- `theme_persistence` 是否提高候选持续性
- `attack_profile` 是否真的提升命中率

只要这几个环节能跑通历史回放，Prism 的可信度会立刻提高一个层级。

### 8.3 第三优先级：拆分 watchlist 和 screener 的研究目标

建议明确两条线：

#### Watchlist 线

更适合加入：

- 基本面趋势
- 预期修正
- 行业景气
- 持仓成本与收益保护
- 仓位滚动规则

#### Screener 线

继续保持：

- 资金
- 题材
- 趋势
- 承接
- 盘中确认

这样两套系统各自会更纯，也更容易优化。

### 8.4 第四优先级：引入组合层风控

哪怕先做轻量版，也建议加入：

- 单票风险预算
- 单主题风险预算
- 行业集中度限制
- 高相关票的同时持有约束
- 连续亏损日自动降仓
- 市场退潮时全局风险缩放

### 8.5 第五优先级：补数据时点与可信度机制

建议所有关键数据都带：

- `as_of`
- `freshness`
- `confidence`
- `fallback_source`

并在评分里显式折减，而不只是文案提醒。

### 8.6 第六优先级：补齐公开仓库可审计性

建议补充或重构：

- `stock-analyzer` 技术分的完整来源文件
- 评分阈值说明
- 样例输入输出
- 历史验证脚本

## 9. 最推荐的落地路线

如果只选最值得马上做的三件事，我建议顺序如下：

### 路线 1：参数标准化

目标：把当前散落在脚本中的规则整理成正式参数字典。

结果：后续所有优化都有统一语言。

### 路线 2：研究验证

目标：用历史样本重放当前规则，输出胜率、收益、回撤、参数敏感性。

结果：把“经验阈值”变成“有证据的阈值”。

### 路线 3：组合化

目标：从单票助手升级成组合级投资系统。

结果：Prism 才真正开始接近专业量化系统。

## 10. 最终判断

Prism 当前已经不是 demo，也不是简单脚本，而是一套具有明确交易哲学的战术决策系统。

它的交易哲学很清楚：

- 不在弱环境里硬做进攻
- 不因为高分就追最热的票
- 不把晨间强势直接当成全天有效
- 不把分析停留在“看多看空”，而是落到执行位和纪律

这是它最有价值的部分。

但要成为真正专业的投资研究系统，Prism 下一阶段必须完成从：

> 经验规则系统

走向：

> 因子化、可回测、可归因、可组合管理的研究系统

当前最现实、最有效的方向，不是再叠更多规则，而是先把现有规则做成：

- 可定义
- 可验证
- 可归因
- 可迭代

只有这样，Prism 的股票分析能力才能从“会看盘”升级到“会证明自己为什么值得信”。
