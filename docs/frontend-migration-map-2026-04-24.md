# Prism 前端迁移对照表

基于远端 `origin/codex/ask-v2` 最新提交 `f655f70` 梳理。

这份对照表回答三件事：

1. 旧控制台页面在新前端里分别落到了哪里
2. 哪些能力已经迁移完成，哪些只是部分落地
3. 现阶段如果切到新前端，会少掉什么

## 一句话结论

这次不是只重写视觉层，而是已经落地了一套可编译的 `Next.js` 新前端骨架。

但它目前仍然是“新决策前端 + 旧 FastAPI API”的并行形态，不是老控制台 1:1 完整替身。

结论可以直接概括为：

- 决策页已经基本迁过去了
- 统一个股页已经成型
- 运维页和参数页还没迁完

## 页面级迁移

### 1. 今日页 `today -> /`

旧页核心：

- 今日一句话结论
- 雷达卡片
- 持仓动作
- 观察池动作
- 风险与变更
- 刷新状态
- 证据摘要和原始入口

新页对应：

- Hero 结论保留
- 4 张 Summary/Radar 卡保留
- `Action Queue` 合并了今日动作，交互上比旧页更强
- 风险提醒保留
- Quick Links 和 Source Cards 保留
- 今日动作状态更新已经接入 API

判断：

- 首页主决策能力已迁移完成
- 信息结构比旧页更清晰
- 但旧页那种“刷新状态条 + 自动刷新 + 更细证据展开层”目前没有完整搬过去

### 2. 问股页 `ask -> Command Bar + /stock/{code}`

旧页核心：

- 搜索框
- 联想建议
- 最近问过
- 示例股票
- 单股结论页
- ask followup 追问
- 从问股页直接加回持仓

新页对应：

- 全局 `Command Bar` 已支持股票联想
- 搜到股票后直接进入 `/stock/{code}`
- `/stock/{code}` 已有统一头部和 4 个 tab

判断：

- 搜索入口已经升级
- 单股结果展示已经升级
- 但旧问股最关键的“追问对话”还没迁
- 最近查询、示例候选、问股页内管理动作也还没迁

这是当前最大的“能力未闭环”之一。

### 3. 持仓页 `watchlist -> /portfolio`

旧页核心：

- Top 3 优先处理
- Follow / Observe 两层展开
- 持仓状态栏
- 刷新状态
- 持仓名单管理
- 添加、归档、恢复
- 刷新日志和过程状态

新页对应：

- 持仓主结论保留
- Summary Cards 保留
- `Priority / Follow / Observe` 三组看板保留
- 焦点标签和来源卡片保留

判断：

- 持仓“看板层”已迁移完成
- 持仓“管理层”还没迁过去
- 也就是说，现在能看，不能完整管

如果你日常很依赖名单增删、归档恢复、刷新状态链路，这一块现在还不够替代老页。

### 4. 观察池 `opportunities -> /discovery`

旧页核心：

- Top 3 继续观察
- 二级分组
- 主线雷达
- 可信度开关
- 质检状态
- 原始数据入口
- 刷新状态

新页对应：

- Summary Cards 保留
- 分组切换保留
- 候选列表保留
- 主线热力保留
- 数据源状态保留

判断：

- 观察池的“主工作面”已迁过去
- 但可信度开关、质检面板、原始数据入口、刷新控制还没完整落地
- 独立批次详情也还没回来

### 5. 复盘页 `review -> /review`

旧页核心：

- 结论区
- 三条校准规则
- 最近结论变化
- 状态栏
- 研究窗口切换
- 来源快照
- 可信度总开关
- 生命周期回放
- 研究拆解和原始入口

新页对应：

- 标题结论保留
- Meta pills 保留
- Summary cards 保留
- Verdict cards 保留
- 校准规则保留
- 变化回放保留
- 对比卡片保留
- 数据源保留

判断：

- 复盘主页面已基本迁移
- 但旧页较深的“研究层”和“可信度开关”没有完整回归
- 独立 detail 页面被折叠回单页

### 6. 参数页 `parameters -> /settings`

旧页核心：

- 参数摘要
- 当前读写文件
- 必填阈值组
- threshold_sets 结构盘点
- 原始 JSON 编辑器
- 保存并同步

新页对应：

- 只有一句“后续补参数编辑”的说明
- 实际页面只展示系统状态、任务和运行记录

判断：

- 这块基本还没迁
- API 封装已经写了，但页面还没接 UI

### 7. 控制台 `dashboard -> /settings`

旧页核心：

- 系统健康
- 失败或阻塞任务
- 手动触发任务
- 飞书开关
- 查看日志
- 运行历史
- 质检总览预览

新页对应：

- 系统健康保留
- 最近运行保留
- 任务列表保留
- 存储路径保留

判断：

- 运维总览有了
- 运维动作还没有
- 旧控制台更像“能操作的后台”，新设置页目前更像“只读状态面板”

## 功能迁移矩阵

| 能力 | 新前端状态 | 备注 |
| --- | --- | --- |
| 全局导航 | 已升级 | 侧边栏 + 移动端导航 |
| 全局搜索 | 已升级 | `Command Bar` 替代旧问股入口 |
| 今日动作勾选 | 已迁移 | 已接 `POST /api/today/actions/decision` |
| 统一股票页 | 已迁移 | 旧多个股票详情入口合并 |
| 响应式布局 | 已升级 | 新前端已支持移动端 |
| 旧问股追问 | 未迁移 | API 在，UI 没接 |
| 最近查询 / 示例候选 | 未迁移 | 旧 ask 页有，新页无 |
| 持仓增删改 | 未迁移 | API 在，UI 没接 |
| 参数编辑 | 未迁移 | API 在，UI 没接 |
| 手动跑任务 | 未迁移 | API 在，UI 没接 |
| 日志预览 | 未迁移 | 旧页有 preview drawer，新页无 |
| 刷新状态条 | 部分迁移 | 只有页面刷新按钮，没有旧式刷新控制条 |
| 可信度总开关 | 未迁移 | 旧 review / opportunities 有，新页无 |
| 独立 detail/batch 页面 | 部分迁移 | 大多折叠回单页或分组页 |

## 现在切过去的收益

- 信息架构明显更统一
- 同一只股票终于有统一入口
- 首页更像“今天该干什么”而不是“今天有什么信息”
- 前端技术栈已经现代化，后续加组件会快很多
- 移动端和键盘流都比旧版强

## 现在切过去的代价

- 会失去一部分老控制台的运维动作
- 会失去旧参数页的可编辑能力
- 会失去 ask followup 追问闭环
- 会失去部分深层证据页和批次页

## 我给你的实际判断

如果目标是“把 Prism 的主决策前台重做出来”，这次重写已经成功了。

如果目标是“今天就把老控制台整套替换掉”，现在还差最后一段后台操作层：

- 参数编辑
- 手动任务触发
- 日志/预览
- 问股追问
- 持仓管理动作

最稳的切换方式不是一次性替换，而是：

1. 先把新前端作为主决策入口
2. 暂时保留老控制台承担运维和配置
3. 把上面 5 个缺口补完后再彻底切换
