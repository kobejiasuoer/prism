# 棱镜 Prism — 页面设计规范

> 交易决策台前端完整重设计方案  
> 版本 1.0 · 2026-04-24

---

## 一、设计理念

棱镜的视觉语言融合 **Bloomberg Terminal 的信息密度** 与 **Linear 的现代极简美学**。
深色优先，减少视觉噪音，让数据和决策信号成为视觉焦点。

核心原则：

1. **信息层级清晰** — 通过背景层级（4层灰阶）和字重区分主次
2. **语义色彩驱动** — 绿/红/黄/蓝 对应 正面/负面/警告/信息，贯穿全局
3. **交易动作色调** — buy/sell/watch/hold/avoid 各有专属色调，一眼识别
4. **克制的装饰** — 无渐变、无阴影、无圆角过度，用边框和间距建立结构

## 二、设计令牌 (Design Tokens)

### 背景层级
| 层级 | 色值 | 用途 |
|------|------|------|
| Primary | `#0a0a0b` | 页面底色 |
| Secondary | `#111113` | 侧边栏、卡片底色 |
| Tertiary | `#18181b` | 输入框、悬浮卡片 |
| Elevated | `#1c1c1f` | 弹窗、浮层 |

### 边框
| 类型 | 色值 | 用途 |
|------|------|------|
| Subtle | `rgba(255,255,255,0.06)` | 默认分割线 |
| Default | `rgba(255,255,255,0.1)` | 交互边框 |
| Strong | `rgba(255,255,255,0.16)` | 高亮边框 |

### 文字
| 层级 | 色值 | 用途 |
|------|------|------|
| Primary | `#f4f4f5` | 标题、关键数据 |
| Secondary | `#a1a1aa` | 正文、描述 |
| Tertiary | `#71717a` | 辅助、标签 |
| Inverse | `#09090b` | 深色文字（浅色按钮上） |

### 语义色
| 语义 | 色值 | 用途 |
|------|------|------|
| Positive | `#22c55e` | 上涨、成功、开启 |
| Negative | `#ef4444` | 下跌、失败、警报 |
| Warning | `#f59e0b` | 观察、等待、注意 |
| Info | `#3b82f6` | 信息、链接、强调 |

### 交易动作色调
| 动作 | 色值 | 含义 |
|------|------|------|
| Buy | `#22c55e` | 加仓 |
| Sell | `#ef4444` | 减仓/清仓 |
| Watch | `#f59e0b` | 观察/试错 |
| Hold | `#3b82f6` | 持有 |
| Avoid | `#71717a` | 回避 |

### 圆角
`sm: 6px` · `md: 8px` · `lg: 12px` · `xl: 16px` · `full: 9999px`

### 字体
- 正文: SF Pro Text / PingFang SC / sans-serif
- 数据: SF Mono / Fira Code / monospace
- 标题: SF Pro Display / PingFang SC / sans-serif

## 三、全局组件

### 侧边栏 (Sidebar)
- 固定宽度 220px，sticky 定位
- 顶部：品牌标识「棱镜 · 交易决策台」
- 搜索框：⌘K 快捷键提示
- 导航项：⌂ 指挥中心 / ◫ 持仓管理 / ◎ 观察池 / ◈ 复盘
- 底部：⚙ 设置 + 系统状态指示灯

### 命令栏 (Command Bar) — ⌘K
- 全局搜索入口，替代原有的独立 Ask 页面
- 支持：股票搜索、页面跳转、快捷操作、AI 对话
- 浮层形式，不占用页面空间

### Badge 徽章
- 药丸形（border-radius: 9999px）
- 背景色为语义色的 10% 透明度
- 边框为语义色的 20% 透明度
- 字号 11px，字重 500

### 指标卡片 (MetricCard)
- 三行结构：标签（11px 大写）→ 数值（28px 粗体）→ 说明（11px）
- 背景 Tertiary，边框 Subtle

## 四、页面设计

### 4.1 指挥中心 (Command Center)
**文件**: `02-command-center.jsx`  
**路径**: `/` (首页)  
**数据源**: `/api/today` + `/api/overview`

**布局结构**:
```
┌─────────────────────────────────────────────┐
│ Hero: 日期 + 阀门状态 + 今日主线 + 约束标签  │
├─────────────────────────────────────────────┤
│ [持仓优先 3] [观察候选 5] [午盘新增 2] [质检 3/3] │
├─────────────────────────────────────────────┤
│ Action Queue (可勾选待办列表)                │
│  ☐ 宁德时代 · 减仓至半仓 · 优先处理          │
│  ☐ 比亚迪 · 止损观察 · 优先处理              │
│  ☐ 中际旭创 · 轻仓试错 · 等触发              │
│  ☑ 寒武纪 · 继续持有 · 已处理                │
├─────────────────────────────────────────────┤
│ 风险提醒 (红/黄警告条)                       │
├──────────────────────┬──────────────────────┤
│ 快速跳转              │ 数据源 (新鲜度指示灯) │
└──────────────────────┴──────────────────────┘
```

**设计要点**:
- Hero 区域是每日决策的「一句话总结」，字号 32px，直接告诉用户今天该做什么
- Action Queue 是核心交互：勾选 = 已处理，左侧色条 = 交易动作色调
- 风险提醒用红/黄背景色的 8% 透明度做底，视觉醒目但不刺眼
- 数据源用绿/黄圆点表示新鲜度，一眼判断数据是否过期

### 4.2 个股档案 (Stock Profile)
**文件**: `03-stock-profile.jsx`  
**路径**: `/stock/:code`  
**数据源**: `/api/watchlist`, `/api/opportunities`, `/api/review`

**核心创新**: 将原来分散在 4 个页面的个股信息合并为一个统一档案页。

**布局结构**:
```
┌─────────────────────────────────────────────┐
│ Header: 名称 + 代码 + 价格 + 涨跌 + 操作按钮 │
├─────────────────────────────────────────────┤
│ [决策] [持仓] [发现] [对话] [历史]  ← Tab 导航 │
├─────────────────────────────────────────────┤
│                                             │
│  Tab 内容区域                                │
│                                             │
└─────────────────────────────────────────────┘
```

**5 个 Tab**:
1. **决策 Tab** — 结论卡片（大字号动作建议）+ 3 张决策卡（仓位/止损/下一步）+ 执行循环（5步流程）+ 判断依据
2. **持仓 Tab** — 元数据行（仓位/优先级/支撑位/阻力位/止损位）+ 盘中触发条件 + 洞察分组
3. **发现 Tab** — 评分指标（综合/质量/一致性）+ 行动计划 + 资金流向 + 洞察标签
4. **对话 Tab** — AI 对话界面，针对该股票的上下文问答
5. **历史 Tab** — 时间线，记录该股票在系统中的所有状态变化

### 4.4 观察池 (Discovery)
**文件**: `05-discovery.jsx`  
**路径**: `/discovery`  
**数据源**: `/api/opportunities`

**布局结构**:
```
┌─────────────────────────────────────────────┐
│ Header + 阀门状态 Badge + [刷新]             │
├─────────────────────────────────────────────┤
│ Pipeline: [全部 12] [早盘进入 5] [继续观察 4] [午盘新增 2] [午盘确认 1] │
├──────────────────────────┬──────────────────┤
│ 候选列表                  │ 进攻阀门信息      │
│ ┌──────────────────────┐ │ 质检状态          │
│ │ 中际旭创 300308  87分 │ │ 主线热力          │
│ │ 光迅科技 002281  82分 │ │ 数据源            │
│ │ ...                  │ │                  │
│ └──────────────────────┘ │                  │
└──────────────────────────┴──────────────────┘
```

**设计要点**:
- Pipeline 阶段选择器是核心导航，底部有 3px 色条指示当前阶段
- 候选行包含：色条 + 名称/代码 + 状态 Badge + Setup + 风险 + 评分 + 涨跌
- 右侧栏信息密度高：阀门状态、质检、主线热力（hot/warm/cool）、数据源
- ThemeCard 用热力 Badge 区分板块活跃度

### 4.5 复盘仪表盘 (Review)
**文件**: `06-review.jsx`  
**路径**: `/review`  
**数据源**: `/api/review`

**布局结构**:
```
┌─────────────────────────────────────────────┐
│ Header: 复盘仪表盘                           │
├─────────────────────────────────────────────┤
│ 环境仪表: [弱环境 -1.2%] [试错环境 +2.8%] [进攻环境 +4.5%] │
├─────────────────────────────────────────────┤
│ 校准规则                                     │
│  ▌弱环境仍未转正 → 少动                      │
│  ▌试错环境已转正 → 轻仓试错                   │
│  ▌进攻环境样本充足 → 等确认                   │
├──────────────────────┬──────────────────────┤
│ 基准 vs 最新切片      │ 变化回放 (Timeline)   │
│ AI 5日净  -0.8→+1.2  │ 04-24 中际旭创 进入   │
│ 弱环境    -2.1→-1.2  │ 04-24 光迅科技 升级   │
│ ...                  │ 04-23 隆基绿能 降级   │
├──────────────────────┴──────────────────────┤
│ 研究面板: [Q1 校正基准] [最新切片]             │
└─────────────────────────────────────────────┘
```

**设计要点**:
- GaugeCard 用中心线 + 单侧色条可视化正负偏移
- 校准规则用左侧 3px 色条 + Badge 标注动作建议
- 对比面板支持 Q1/最新切片 切换
- 变化回放用彩色圆点区分事件类型（进入/升级/降级/退出/移交）

### 4.6 设置 (Settings)
**文件**: `07-settings.jsx`  
**路径**: `/settings`  
**数据源**: `/api/parameters`, `/api/tasks`, `/api/health`

**三个子面板**:

1. **任务管理** — 5 个数据采集/分析任务的执行与排期
   - 任务卡片：名称 + 状态 Badge + 执行按钮 + 进度条
   - 顶部 4 个汇总指标

2. **参数配置** — 策略参数的可视化编辑
   - 双栏布局：阀门控制 + 环境评级 / 扫描策略 + 数据刷新
   - 支持 toggle、select、number 三种输入类型
   - 底部保留原始 JSON 编辑器入口

3. **系统状态** — 核心服务健康监控
   - 顶部 Uptime Banner（可用率 + 状态摘要）
   - 服务列表：后端/采集引擎/AI服务/行情数据源/本地存储
   - 最近日志流（等宽字体，带时间戳和级别着色）

## 五、交互模式

### 导航
- 侧边栏常驻，点击切换页面
- ⌘K 命令栏覆盖在当前页面上方
- 个股档案通过点击任意股票名称/代码进入

### 数据刷新
- 每个页面顶部有「刷新」按钮
- 数据源面板显示各数据的最后更新时间和新鲜度
- 自动刷新间隔可在设置中配置

### 状态反馈
- 任务运行中显示进度条
- 数据新鲜度用绿/黄圆点指示
- 系统异常用红色 Badge + 警告条提示
