# Prism Working Anchor

Date: 2026-05-23

This is the short, reusable anchor for resumed work. Read this first when context is long.

## North Star

Prism should help users make more stable, executable daily decisions because they rely on Prism.

## What Must Not Drift

- Do not confuse fresh data with authoritative data.
- Do not let display-only data become live or formal decision input.
- Do not claim formal readiness without calendar, benchmark, adjustment, and execution support.
- Do not expand scope into product fantasies, ML, or unrelated rewrites while data grounding is still incomplete.

## Three Stages

### Stage 1: Data Governance

Make source authority explicit, queryable, and visible.

Focus:

- dataset registry
- authority metadata
- fallback visibility
- schema and semantic checks
- readiness display

### Stage 2: Formal Daily Sources

Freeze the daily research backbone.

Focus:

- trade calendar
- benchmark daily series
- adjustment factors
- price limits
- formal label eligibility

### Stage 3: Execution Reality

Model what can actually be traded.

Focus:

- suspend status
- limit-up / limit-down constraints
- failed order outcomes
- partial fill and cost realism

## Current Working Rule

Only one stage is active at a time. Keep the next stage visible, but do not work all three at once.

## Session Status Template

Use this exact format when handing work off:

```text
Current stage:
Done:
In progress:
Blocked:
Next:
Decision needed:
```

## Good Finish Criteria

Before closing a card, ask:

1. Does this move Prism closer to trusted daily decisions?
2. Does it reduce source ambiguity?
3. Does it make the next session easier to resume?
4. Did we avoid scope drift?
